The variables

Introduction

A variable refers to the fact of assigning a name or identifier to information: by naming them, one can manipulate this information much more easily.

The advantage of using variables is that you can write programs that are valid for values that vary: you can change the value of the variables, and the program will always run the same way and do the same types of calculations regardless of the values manipulated. Variables play a role similar to the unknowns in a mathematical equation.

Definitions

Definition : Notion of variable

A variable is characterized by :

  • an identifier : it can contain letters, numbers, underlined blanks but it cannot start with a number. Lower case and upper case are differentiated. It is also unique.
  • a type : is an information about the content of the variable which tells the python interpreter how to handle this information.

Definition : Constant variables

Constants are the opposite of variables, they are all numerical values, strings, ..., everything that is not designated by a name. Constants have a type but no identifier.

Fondamental : Dynamic typing

The typing in Python is dynamic : the type is not explicitly specified, it is implicitly linked to the manipulated information. For example, writing x=3.4 does not specify the type of the variable x but it is implicit because x receives a real value : x is of type float in Python.

Unlike languages such as C, there is no need to declare a variable to indicate that it exists, it is enough to assign a value to it. The type of the variable will be defined by the type of the constant assigned to it. The type of a variable can change, it always corresponds to the type of the last assignment.

Data types in Python

Integers and real numbers

There are two types of numbers in Python, real numbers float and integers int. The instruction x=3 creates a variable of type int initialized to 3 while y=3.0 creates a variable of type float initialized to 3.0.

Immutable type (or immutable)

A variable of immutable type cannot be modified. An operation on a variable of this type necessarily leads to the creation of another variable of the same type, even if the latter is temporary.

In other words, the simple instruction x+=3 which consists in adding to the variable x the value 3 creates a second variable whose value is that of x increased by 3 then to copy its contents in that of the variable x. Numbers are immutable types just like strings and tuples which are arrays of objects. It is not possible to modify a variable of this type, it is necessary to recreate another one of the same type which will integrate the modification.

None type

Python provides a None type to mean that a variable contains nothing. The variable is of type None and is equal to None.

The functions in Python language

Concept

Introduction

Functions are small programs that allow to modularize an algorithm into elementary tasks.

Advantages :

  • Better readability of the code: the final program is thus easier to understand ;
  • Allows better debugging: it is easier to isolate an error if one occurs: just test one by one or the functions to determine which one returns a bad result;
  • Avoid and prevent code redundancy.

Definition : Function

A function is a part of a program - or subprogram - that operates independently of the rest of the program. It receives a list of parameters and returns a result. The body of the function is any statement in the program that is executed if the function is called.

Fondamental : Declaration of a fonction

   1

   2
   instruction 1  

   3
   ...   

   4

   5
   return res_1, ..., res_n
  • function_name is the name of the function, it follows the same rules as the variable names.
  • par_1 to par_n are the names of the parameters.
  • res_1 to res_n are the results returned by the function.
  • The instructions associated with a function must be indented with respect to the def keyword.
  • If there is no result, the return statement is optional or can be used alone without being followed by a value or a variable. This statement can appear several times in the function code but only one of them will be executed. From that moment on, all other statements in the function will be ignored.
  • The global instruction allows you to extend (and thus modify) the scope of a variable to the enclosing module; by default the scope of a variable defined in a function is local.

Scope of variables

Generalities

Defining a function means defining a new namespace, so all internal declarations of a function are not visible to other functions. In this case we speak of local scope to designate this behavior.

A function is defined within an enclosing space called a module, which has a global scope.

These two types are linked as follows:

  • The enclosing module defines a global scope, i.e. a space where the created variables reside (by assignment).
  • Each function call (in the module) defines a new local scope, i.e. a self-contained space for names that does not conflict with the enclosing space.
  • Assigned names are local, unless explicitly declared otherwise by the global statement.
  • All remaining names are either global (i.e. defined in the enclosing module) or internal (i.e. names pre-defined by python).

Fondamental : 3-span rules L, G and I

The reference to variable names triggers the search within three levels of scope, local (L), global (G) and finally internal (I).

Name assignment creates or modifies local names by default

The declaration by the global statement allows the names to be attached to the enclosing module (namespace).

Definition : Local variable

Parameters order

During the call, the name of the parameters does not intervene anymore, assuming that each parameter receives for value the one that has the same position as it when the function is called. It is however possible to change this order by specifying which parameter should receive which value.

   1
x_1, ..., x_n = function_name (param_1 = value_1, ..., param_n = value_n)

Where function_name is the name of the function, param_1 to param_n are the name of the parameters, value_1 to value_n are the parameters values.

With this syntax, the writing order does not matter. The value value_i will always be assigned to param_i. The variables x_1 to x_n receive the results returned by the function.

The order of the results cannot be changed. If there are several results returned, it is impossible to choose which ones to keep: either all or none.

This possibility is interesting especially when there are many default parameters and only the value of one of the last parameters must be changed.

Parameters modifications

Parameters of non mutable and modifiable types behave in different ways inside a function.

These parameters are manipulated in the body of the function and sometimes even modified. Depending on the type of parameter, these modifications have repercussions outside the function. Immutable types cannot be modified and this remains true. When a function accepts a parameter of an immutable type, it only receives a copy of its value.

It can therefore modify this parameter without affecting the variable or value used when the function was called. This is called parameter passing by value.

On the other hand, any modification of a variable of a modifiable type inside a function is reflected in the variable that was used when the function was called. This second type of passing is called passing by address.

The variable nb is of a non mutable type. Its value has been copied into the parameter n of the function elements_sum

Remarque

In practice, parameter passing in Python is very similar to that of C language :

  • The non-modifiable (mutable) parameters correspond to the passage by reference of C. In fact the passage is done by reference, but since it is impossible to change non-modifiable objects in place, it amounts to making a copy.
  • The modifiable (mutable) parameters, such as lists, dictionaries ... correspond to the C address passage.

A recursive function is a function that calls itself.
